## Overview

DevCycle 11 focuses on preparing for the version 0.0.10 release by implementing a key user-requested feature: the ability to perform songs directly from the Edit Song Details page. This enhancement bridges the gap between song management and venue performance tracking, allowing users to quickly initiate performances while viewing song details from venue contexts.

## Current Work Items

**Status:** 📋 Open  
**Priority:** High  
**Description:** Add a "Perform Song" button to the Edit Song Details page that allows users to perform the song at the current venue context. The system should automatically handle visit management - using an active visit if one exists, or creating a new active visit if needed.

#### Technical Implementation Details
**Target Page:** EditSongVenueInfoFragment (not EditSongFragment) - this is the "Edit Song Details" page accessible from venue contexts
**UI Integration:** Added full-width "Perform Song" button with music note icon above the Save/Delete buttons
**Venue Context:** Automatically detected from existing navigation arguments and ViewModel data structure
**Visit Management:**
- Checks for existing active visit using `repository.getActiveVisitForVenue(venueId)`
- Creates new active visit if none exists with `repository.insertVisit(newVisit)`
- New visits are automatically marked as active (`isActive = true`)
**Performance Recording:**
- Creates Performance entity with visit association, song, key adjustment, and timestamp
- Uses current key adjustment from UI (including "Unknown" handling)
- Records performance using `repository.insertPerformance(performance)`
- Updates performance count display automatically after successful recording
**User Feedback:** Toast messages for success/failure with automatic result clearing
**Error Handling:** Try-catch blocks with proper loading state management and user notification
**Integration:** Builds on existing repository methods and entity relationships
